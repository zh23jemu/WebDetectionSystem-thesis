# 木材表面缺陷智能检测系统文档

## 1. 项目简介

本项目是一个基于 Python 的 Web 木材表面缺陷智能检测系统，面向木材分级、质检和销售管理场景。系统以 YOLO 模型为核心，对上传图片或摄像头采集图片进行缺陷识别，并将检测结果进一步映射为板材等级，同时联动库存、历史记录、日志报表和销售管理等业务模块。

系统当前识别的缺陷类别主要包括：

- 裂纹（Crack）
- 死节（Dead Knot）
- 活节（Live Knot）

对应的分级规则为：

- `A级`：无缺陷
- `B级`：存在活节等较轻缺陷
- `C级`：存在裂纹、死节等严重缺陷

## 2. 项目功能

### 2.1 用户与登录

- 支持普通用户与管理员双角色登录
- 登录时校验账号角色与登录入口是否一致
- 支持安全退出
- 记录用户登录时间、退出时间与工作时长
- 支持个人中心修改密码

### 2.2 木材缺陷检测

- 支持本地图片上传检测
- 支持摄像头拍照检测
- 支持单张或多张图片批量检测
- 基于 YOLO 模型进行目标检测
- 将英文缺陷类别自动映射为中文结果
- 根据缺陷类型自动判定 `A级 / B级 / C级`
- 检测后自动保存结果图到 `static/results`

### 2.3 检测历史管理

- 自动记录检测文件名、检测结果图、缺陷类型、置信度、检测时间、用户 ID、等级判定
- 支持分页查看历史记录
- 支持按缺陷类型搜索
- 支持删除检测记录
- 支持历史结果大图预览

### 2.4 日志与报表

- 展示今日检测量
- 统计累计工作时长
- 统计缺陷类别占比
- 展示近 7 天检测趋势图
- 支持将检测数据导出为 Excel

### 2.5 库存与销售管理

- 检测完成后自动按等级写入库存
- 支持查看各等级库存数量
- 支持新建销售订单
- 销售出库时自动扣减库存
- 支持查看销售记录

### 2.6 知识库与帮助中心

- 提供常见木材缺陷图谱
- 包含活节、死节、裂纹的定义、特征与等级说明
- 可作为新员工或新用户的学习参考页

### 2.7 管理员功能

- 用户管理：新增用户、删除用户
- 系统设置：调整检测置信度阈值
- 模型管理：上传新的 `.pt` 权重文件并切换当前检测模型

## 3. 技术栈

### 3.1 后端

- Werkzeug

- SQLite

- `wood_system.db`

- Ultralytics YOLO

### 3.4 前端

- Jinja2 模板
- Bootstrap 5

- Viewer.js

- pandas
- openpyxl

说明：代码中使用了 `pandas` 和 `openpyxl` 导出 Excel，但这两个依赖当前没有写入 `requirements.txt`，运行前建议一并安装。

## 4. 核心目录与文件说明

- `app.py`：Flask 应用入口，包含主要路由与业务逻辑
- `config.py`：系统配置，包括数据库路径、上传目录、结果目录、模型路径
- `models.py`：数据库模型定义
- `extensions.py`：Flask 扩展初始化
- `utils/detection.py`：YOLO 模型加载与图片检测逻辑
- `templates/`：前端页面模板
- `static/img/`：帮助中心缺陷示例图片
- `static/results/`：检测结果图片输出目录
- `best.pt`：默认检测模型权重
- `yolov8n.pt`：项目内额外模型文件
- `wood_system.db`：SQLite 数据库文件

## 5. 如何运行

以下命令基于 Windows 环境，并按照项目要求使用本地虚拟环境 `.venv`。

### 5.1 进入项目目录

```powershell
cd C:\Coding\WebDetectionSystem-thesis\WebDetectionSystem
```

### 5.2 创建虚拟环境

如果项目下还没有 `.venv`，先创建：

```powershell
python -m venv .venv
```

### 5.3 安装依赖

先安装 `requirements.txt` 中的依赖：

```powershell
.venv\Scripts\pip.exe install -r requirements.txt
```

再补充安装导出 Excel 所需依赖：

```powershell
.venv\Scripts\pip.exe install pandas openpyxl
```

### 5.4 启动项目

```powershell
.venv\Scripts\python.exe app.py
```

启动后访问：

- [http://127.0.0.1:5000](http://127.0.0.1:5000)

## 6. 初始化行为

项目首次启动时会自动执行以下初始化操作：

- 自动创建数据库表
- 如果不存在管理员账号，则自动创建默认管理员
- 如果库存表为空，则自动初始化 `A级`、`B级`、`C级` 三类库存
- 自动创建上传目录和结果目录

默认管理员账号为：

- 用户名：`admin`
- 密码：`123456`

## 7. 运行条件与注意事项

- 默认模型文件为项目根目录下的 `best.pt`
- 若系统设置中切换了模型，后续检测会优先使用新上传的 `.pt` 文件
- 摄像头检测依赖浏览器权限，首次使用时需要允许访问摄像头
- 检测结果图片会持续保存在 `static/results` 中
- 项目当前使用 SQLite，适合课程设计、原型验证和单机部署
- 若用于多人并发或生产环境，建议后续迁移到 MySQL/PostgreSQL，并补充分离式部署方案

## 8. 总结

该项目本质上是一个“木材缺陷检测 + 分级入库 + 历史追溯 + 销售出库 + 报表统计”的一体化 Web 系统。它将深度学习检测能力和传统业务管理流程结合起来，既能完成图像识别任务，也能支撑基础的检测业务闭环，适合作为毕业设计或教学演示项目。
